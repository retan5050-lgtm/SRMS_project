Student Records Management System (SRMS)
This project is a simple Student Records Management System developed in Java. It provides a Command Line Interface (CLI) 
to add, update, delete, and view student data, along with functionality for persistence (saving and loading data to/from a file).

 Key Features
• Student Management:
• Add a new student (ID, Name, GPA, and Department).
• Update existing student details (Name, GPA, and Department).
• Remove a student by their ID.

 Features:
• Generate and print a comprehensive report listing all current student records.
• Data Persistence:
• Save student data permanently to a file (students).
• Load (retrieve) student data from the file upon program startup.
• Automatic Saving (AutoSave):
• Utilizes multi-threading to automatically save the data in the background every 5 minutes.
• Object-Oriented Programming (OOP):
• Applies core OOP concepts using classes like Person, Student, StudentManager, and FileHandler.


Technologies Used:
 • Java 
 • NetBeans IDE
 • CSV
 • Multithreading 

How to run the Program:
1. Make sure you have copied all the code correctly into their respective files.
2. In NetBeans, right-click on the Main.java file (or the entire project) and select Run File .
3. The program will execute, and the menu will appear in the Output Window at the bottom of the NetBeans screen.

Conclusion:

This project demonstrates the application of Java OOP concepts, including inheritance, encapsulation, modular design, file handling, and multithreading
The system provides a complete and functional student record management solution with persistent storage and real-time background saving, making it suitable for academic and practical use.
